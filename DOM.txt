DOM
stands for document object model
In essence, browser takes the html code, and parse it (breaks it apart),
then builds a DOM tree with the html code. Each tag, attribute, content, etc. becomes a node OBJECT in memory that has properties that JS can access
EX: <div class="blue">Hello</div>
In DOM, div becomes an OBJECT like below:
{
  tagName: 'DIV',
  classList: ['blue'],
  innerText: 'Hello',
  style: { ... },
  ...
}

EX: <div id="box">
     <p>Hello</p>
    </div>

VERY POWERFUL!!
allows us to use css selectors to access DOM elements w/ javascript
EX: document.querySelector('p');
//specify a css selector as a string and returns the first element that matches that selector.
//example above would return the first paragraph in the document.

.getElementById
EX: document.getElementById('bio').innerHTML = 'The description';

*IMPORTANT* The difference between .querySelector and .getElementById is that .querySelector can select ANY CSS selectors.
//document.querySelector("#box");     // by ID
document.querySelector(".item");    // by class

-----------------------------------------------------------------
.getElementsByClassName & .getElementsByTagName
return an array of all the elements with that class/tag name. Must use bracket to access the correct element

EX: document.getElementsByClassName('student')[0].innerHTML = 'Not yet registered';
// Set first element of .student class as 'Not yet registered'

EX: document.getElementsByTagName('li')[1].innerHTML = 'Cedric Diggory`;
// Set second <li> tag as 'Cedric Diggory'

*IMPORTANT*
let blueElement = document.querySelector('.blue');
blueElement.style.backgroundColor = 'blue';
browser turns html codes into nodes of objects called DOM.
document.querySelector('.blue') access the first node that matches .blue and returns the OBJECT and assign it to blueElement.
Since blueElement is a variable that reference the object, we can use the variable to access the object's properties.

Also, the ".style" automatically apply css properties that you would normally add in stylesheet.
However, since JS cannot read hyphen, you must use camel style. background-color becomes backgroundColor

create/inserting new element via JS
.createElement()
create new element based on the tag passed in as argument
EX: let paragraph = document.createElement('p');
JS creates this element object and assigned it to variable paragraph. However, it's NOT in the document because
we never specified where this <p> is going to be placed at. We simply created this element.

EX: paragraph.id = 'info'; 
    paragraph.innerHTML = 'The text inside the paragraph';
next we assign properties to this element like content, attributes, etc.

In order to add the element to the document, we need to append it
EX: document.body.appendChild(paragraph);
Here we tell the browser to append the paragraph element to body
*IMPORTANT* browser append the element as the last child of that parent.

EX: document.querySelector('#italy-attractions').appendChild(newAttraction);
If you need to put a new element inside the document somewhere specific, you must first get that element object
using .querySelector, getElementById, etc then call the appendChild property on that element object.
-----------------------------------------------------------------
Hide an element
EX: let paragraph = document.querySelector('p');
    document.body.removeChild(paragraph);
First we get the first p element via querySelector and reference that node object by variable paragraph
Next we remove that element via removeChild from the body (in this case)
*IMPORTANT* removeChild only works if the element you're removing is a direct child of the parent you're calling from.
In the example above, that code only works if the html look like this:
<body>
  <p>Hello</p>
</body>

If it looked like the structure below, the code will not work
<body>
  <div>
    <p>Hello</p>
  </div>
</body>

.onclick
The onclick property allows you to assign a function to run on when click event happens on an element. 
let element = document.querySelector('button');

function turnButtonRed(){
  // Add code to turn button red
  this.style.backgroundColor = 'red';
  this.style.color = 'white';
  this.innerHTML = 'Red Button';
}

element.onclick = turnButtonRed;
Here, element is an object and onclick is a property of the object. 
Before we set the fuction to onclick, onclick is set to null.
Think of it like this: element = {onclick: null, ......}
We then set turnButtonRed function to onclick, so now the object looks like
{onclick: turnButtonRed(), ......}

Events at the simplest explanation is something that happened in the browser.
Webpages just sit there....waiting until something happens. That something can be mouse moving, clicking, keyboard typing, etc.
When something happens on the webpage, it is called an event. 

Event target and event handler
We can have a DOM element (which is an object) listen for a specific event that is detected on itself, and then execute a block of code. 
That DOM element is called the event target, and the block of code that is executed is called the event handler. 

EX: let eventTarget = document.getElementById('targetElement');

eventTarget.addEventListener('click', function() {

});

The addEventListenter is a method on the event target object from DOM.
It takes in 2 arguments: a event name (what kind of event) and a function that will execute when the event is triggered on the event target.

*IMPORTANT* Instead of using an anonymous function, we can also create a function and pass it as an argument later.

EX:
function eventHandlerFunction() {
  // this block of code will run when click event happens
}

eventTarget.addEventListener('click', eventHandlerFunction);

-----------------------------------------------------------------

.on
Another way to add event handler other than .addEventListener() is to add "on" to the beginning of event type name. 

EX: eventTarget.onclick = eventHandlerFunction;

In the above code, we want to add an event handler to eventTarget with the event being a mouse click. We use on infront of the click type like so .onclick and set it to the function that will be executed.
onclick is actually a DOM property. Developers of DOM made MOST event types as object properties so you can just add 'on' in front of the event type to make it match the DOM property. 

*IMPORTANT* one limitation of 'on' is that it can only register one function to an event target, while .addEventListener can register multiple functions at once. 

element.onclick = fn1;
element.onclick = fn2;

In the above, fn1 will be overwritten by fn2

element.addEventListener('click', fn1);
element.addEventListener('click', fn2);

In the above, both fn1 and fn2 will execute when click is detected on event target. 

-----------------------------------------------------------------

Removing an eventlistener.
Sometimes you want to remove an eventlistener after an event has happened to prevent the user from keeping
activating the event over and over again.
EX: eventTarget.removeEventListener('click', eventHandlerFunction);

EX:
  function showFortune(){
  fortune.innerHTML = fortuneSelector();
  button.innerHTML = "Come back tomorrow!";
  button.style.cursor = "default";
  button.removeEventListener('click', showFortune);
}

button.addEventListener('click', showFortune);

Here we add the removeEventListener inside the showFortune itself. So when showFortune runs, the last step
it will do is to remove the Eventlistener.

-----------------------------------------------------------------

When you trigger an event, JS makes the event into an object with its own properties and methods.
You can then use the event object properties and methods

EX: 
function eventHandlerFunction(event){
   console.log(event.timeStamp);
}

eventTarget.addEventListener('click', eventHandlerFunction);

Here the click event is passed into the eventHandlerFunction as an object automatically by JS
So when click is trigger, the object pass into the function, and a click object property called .timeStamp is called.

*IMPORTANT*, the "event" word is a dummy variable. JS simply pass the event object as the FIRST parameter.
So if you have eventHandlerFunction(obj1, obj2, event), the event object is passed into obj1. The word "event" is irrelevant.

Best Analogy: You click a button → the browser hands your function a report. That report is the event Object.

That report (event object) says:
“This was a click”
“It happened at this time”
“It happened on this element”
“Mouse was here”

-----------------------------------------------------------------

One of the properties of an event object is another object called target. (remember, an object can have another object as property).

event.target

It returns "The exact DOM element that triggered the event"

-----------------------------------------------------------------

Forms are a great way to collect user input data and sent the information to another location like another page, or
to the backend database.

EX: 
<form action="/example.html" method="POST">
</form>

Above is the basic skeleton of a form. The action determines where the information is sent off to and retrieved and method is an http verb
telling the browser how it should be sent (where you want to get data or post new data).
*IMPORTANT* <form> itself is like <div>, it has no physical look on its own. You need to put other elements like <h1>, <p>, etc
inside it to show up on a page.

-----------------------------------------------------------------

<input>

This is how user actually input data.

EX:
<form action="/example.html" method="POST">
  <input type="text" name="first-text-field">
</form>

In the above, <form> has no visual on the screen (much like <div>)
As soon as we add <input>, browser will but a "textbox" on the screen so user can input data.
The type tells what kind of data the user will input. The type also change the input box.
For example, if we change type="radio", the browser will replace the textbox with a radio circle to click on.
The name is the "key" that browser will associate with our input when it sends the information off. Without the name,
the browser will not send data.
*IMPORTANT* Notice that <input> has no closing tag.

-----------------------------------------------------------------

<label>

Label help explain to users what the input is for, especially sites with many input boxes

EX:
<form action="/example.html" method="POST">
  <label for="meal">What do you want to eat?</label>
  <input type="text" name="food" id="meal">
</form>

Here, the label is next to the input box.
Labels are by default, inline elements. In other words, they are next to the input box.
You can use CSS to change label to block level if you want label {display: block}
It is crucial to associate label with the correct input box. In label, we use "for" and
that value is the same as "id" in input.

-----------------------------------------------------------------

<type="password">

The password type replaces input text with another character such as asterisk or dot.

EX: 
<form>
  <label for="user-password">Password: </label>
  <input type="password" id="user-password" name="user-password">
</form>

The <input> still create a textbox but when user type into the box, the characters are asterisk. 

-----------------------------------------------------------------

<type="number">

The number type allows user to input numbers, which can be used to do calculations on the backend.

EX:
<form>
  <label for="years"> Years of experience: </label>
  <input id="years" name="years" type="number" step="1">
</form>

The step creates an up/down arrow to increase/decrease by the value in step. In the above code, the up/down arrow
increase/decrease by 1.

-----------------------------------------------------------------

<type="range">

This creates a slider between a range of numbers, which limits the user to a certain number of inputs.

EX:
<form>
  <label for="volume"> Volume Control</label>
  <input id="volume" name="volume" type="range" min="0" max="100" step="1">
</form>

-----------------------------------------------------------------

<type="checkbox">

Allow the user to choose multiple options

EX:
<form>
  <p>Choose your pizza toppings:</p>
  <label for="cheese">Extra cheese</label>
  <input id="cheese" name="topping" type="checkbox" value="cheese">
  <br>
  <label for="pepperoni">Pepperoni</label>
  <input id="pepperoni" name="topping" type="checkbox" value="pepperoni">
  <br>
  <label for="anchovy">Anchovy</label>
  <input id="anchovy" name="topping" type="checkbox" value="anchovy">
</form>

*IMPORTANT* The value tells what value to sent if the checkbox is checked. By having a value there does NOT mean
the value will be sent, it just tells the browser if the checkbox is checked, then this is the value that will be sent.
Also, notice that name above are all the same. Names are not like id. It does NOT need to be unique.
The name acts as a key in key-value pairs (with value being the value pair).
So when you check multiple boxes: what is sent looks like topping=cheese&pepperoni&anchovy

-----------------------------------------------------------------

<type="radio">

Unlike checkbox where user can click on multiple options, a radio only allows user to click on one option.
This is important for things like agree/disagree on conditions (for example)

EX:
<form>
  <p>What is sum of 1 + 1?</p>
  <input type="radio" id="two" name="answer" value="2">
  <label for="two">2</label>
  <br>
  <input type="radio" id="eleven" name="answer" value="11">
  <label for="eleven">11</label>
</form>

Dropdown list

EX:
<form>
  <label for="lunch">What's for lunch?</label>
  <select id="lunch" name="lunch">
    <option value="pizza">Pizza</option>
    <option value="curry">Curry</option>

-----------------------------------------------------------------

Datalist input

Unlike <section> dropdown which restrict the user to only the dropdown choices, datalist gave a dropdown list BUT also
allow the user to type in answer.

<form>
  <label for="city">Ideal city to visit?</label>
  <input type="text" list="cities" id="city" name="city">

  <datalist id="cities">
    <option value="New York City"></option>
    <option value="Tokyo"></option>
    <option value="Barcelona"></option>
    <option value="Mexico City"></option>
    <option value="Melbourne"></option>
    <option value="Other"></option>  
  </datalist>
</form>

In the above, input type is text. Notice that there is a new category in <input> called list.
It matches with the id in datalist.

-----------------------------------------------------------------

textarea

If a user need to write more information, then <input type='text'> is not efficient.
For that we can use <textarea>

<form>
  <label for="blog">New Blog Post: </label>
  <br>
  <textarea id="blog" name="blog" rows="5" cols="30">
  </textarea>
</form>

In the above, rows and column indicate how big the text box will be. It does NOT restrict how much you can write.
In other words, if you write more, the browser will automatically create a scrollbar.
But the box size will stay as 5 rows and 30 column wide.

-----------------------------------------------------------------

<type='submit'>

To submit a form, html <input> has another type called 'submit'

EX:
<form>
  <input type="submit" value="Send">
</form>

Here the value means what is actually written on the submit button. In this case, the button reads send.

-----------------------------------------------------------------

required

You can use the word required to make certain form not optional such as name, SSN, etc.
User must fill them out to submit the form.

EX:
<input type="number" name="guess" id="guess" required>

-----------------------------------------------------------------

Set a minimum/maximum value

<input type="number" name="guess" id="guess" required min='1' max='10'>

The above code sets a min and max number value.

-----------------------------------------------------------------

Set min/max value for text length

<input id="summary" name="summary" type="text" minlength="5" maxlength="250" required>

-----------------------------------------------------------------

pattern matching

what if you want the user to put into the textbox a specific format like 14 or 16 digit credit card number?

EX:
<form action="/example.html" method="POST">
  <label for="payment">Credit Card Number (no spaces):</label>
  <br>
  <input id="payment" name="payment" type="text" required pattern="[0-9]{14,16}">
  <input type="submit" value="Submit">
</form>

Here pattern allows you to input any javascript regular expression (regex).
In the above, we tell the browser to only allow numbers 0 to 9 and the length must be 14 to 16 digits
